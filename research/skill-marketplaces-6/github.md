## Executive Summary

Round 6 reveals a **massive explosion** in the agent skills ecosystem since prior rounds. The space has evolved from niche Claude Code plugins into a multi-platform ecosystem with 11,000+ star repos, enterprise-grade security tooling, Bitcoin Lightning micropayments, on-chain audit registries, and formal academic frameworks for skill composition. 4. **SkillWeave** published a formal academic framework for governed, cross-boundary skill composition (submitted to IEEE)
5. **skill-loop** introduced YAML-based **skill orchestration with LLM routing** between agents
6. The ecosystem now spans 15+ supported platforms (Claude Code, Codex, Gemini CLI, Cursor, VS Code, Copilot, Manus, OpenCode, Amp, Roo Code, etc.)

## Detailed Analysis of Top New Repos

### 1. SkillX (nextlevelbuilder/skillx) — 34 stars

**The most complete open-source skill marketplace implementation found.**

- **What it is:** Full-stack marketplace combining web UI, CLI tool (`skillx-sh` on npm), and hybrid search engine
- **Architecture:**
  - Frontend: React Router v7 + Tailwind v4 (dark theme, mint accent)
  - Backend: Cloudflare Workers + D1 (SQLite)
  - Search: **Hybrid semantic + keyword** using Vectorize (bge-base-en-v1.5 embeddings) + FTS5 + Reciprocal Rank Fusion
  - Auth: Better Auth + GitHub OAuth
  - Storage: KV (cache), R2 (assets)
- **Key Features:**
  - 500+ skills browsable with ratings (0-10), reviews, usage stats
  - Leaderboard sorted by quality metrics
  - Favorites for personalized search recommendations
  - CLI for programmatic access: `skillx search`, `skillx use`, `skillx report`
  - Claude Code Plugin integration: `/plugin marketplace add nextlevelbuilder/skillx`
  - API key management for CLI access
  - Execution reporting with outcome tracking
- **Monetization Model:** Supports `is_paid` filter in search API — paid vs free skill distinction
- **Search Performance:** <800ms p95 latency
- **Why It Matters:** First open-source marketplace with production-ready search infrastructure, user auth, ratings system, and CLI — the closest thing to an "npm for skills"

### 2. Trail of Bits Skills Marketplace (trailofbits/skills) — 3,589 stars

**The gold standard for domain-specific security skill marketplaces.**

- **What it is:** Claude Code plugin marketplace from the leading security research firm
- **Plugin Categories (30+ plugins):**
  - **Smart Contract Security:** Vulnerability scanners for 6 blockchains, entry point analyzers
  - **Code Auditing:** GitHub Actions auditor, Semgrep rule creators, supply chain risk auditor, variant analysis
  - **Malware Analysis:** YARA rule authoring
  - **Verification:** Constant-time analysis, property-based testing, spec-to-code compliance
  - **Reverse Engineering:** DWARF format expert
  - **Mobile Security:** Firebase APK scanner
  - **Development:** Modern Python, second-opinion reviews via external LLMs
- **Trophy Case:** Real bugs found using skills (e.g., timing side-channel in ML-DSA signing in RustCrypto)
- **Distribution:** Via Claude Code plugin marketplace: `/plugin marketplace add trailofbits/skills`
- **Why It Matters:** Proves the marketplace model works for **high-stakes professional domains**. The trophy case validates that skill-enhanced agents find real vulnerabilities.

### 3. SquidBay (squidbay/squidbay) — 2 stars

**First marketplace where AI agents pay AI agents, using Bitcoin Lightning.**

- **Core Innovation — Three-tier pricing model:**

  | Tier | Model | What You Get |
  |------|-------|-------------|
  | Remote Execution | Rent | Pay-per-use API call |
  | Skill File | Learn | Blueprint/instructions |
  | Full Package | Own | Complete source code |

- **Payment System:**
  - Bitcoin Lightning for instant, global, permissionless payments
  - **2% platform fee** — 98% goes to seller
  - No subscriptions, no lock-in
- **Agent Identity & Reputation:**
  - Verified identity tiers: Unverified → A2A Verified (green check) → X Verified (gold badge)
  - Agent names **locked forever** — cannot rename to dodge bad reviews
  - Real reviews from real transactions
  - Soft deletes only — all history preserved permanently
- **Three Usage Modes:**
  1. Local agents with Lightning wallets (fully autonomous)
  2. Cloud AI with local runtime (Claude Code, Codex)
  3. Humans on the website
- **Tech Stack:** Node.js/Express + SQLite (sql.js) + A2A JSON-RPC protocol + Claude-powered chatbot
- **Live at:** squidbay.io
- **Why It Matters:** First working implementation of **agent-to-agent commerce**. The rent/learn/own pricing model is novel and could become an industry pattern. The permanent identity + reputation system addresses trust in ways centralized marketplaces cannot.

### 4. skill-loop (takumiyoshikawa/skill-loop) — 18 stars

**First dedicated skill orchestration tool — chains skills with LLM-based routing.**

- **What it is:** Go-based CLI that chains multiple coding-agent skills in a loop workflow
- **How it works:**
  1. Reads YAML config defining skills and routing rules
  2. Invokes configured agent (claude, codex, or opencode) with each skill
  3. A **separate LLM router** evaluates skill stdout against route criteria
  4. Loop continues until `done: true` or `max_iterations` reached
- **Key Features:**
  - Multi-agent support: Claude Code, Codex CLI, OpenCode CLI
  - YAML-based workflow definitions with routing criteria
  - Tmux-backed execution with session management
  - **Cron scheduling** for periodic workflows
  - Built-in React dashboard for monitoring sessions
  - `--blocked` state for human-in-the-loop decision points
- **Installation:** Via Homebrew or `go install`
- **Example Workflow:**
  ```yaml
  skills:
    1-impl:
      agent: { runtime: claude, model: claude-sonnet-4-6 }
      next: [{ id: send-review, skill: 2-review }]
    2-review:
      agent: { runtime: codex, model: gpt-5.4 }
      next:
        - { id: approve, criteria: "quality sufficient", done: true }
        - { id: rework, criteria: "issues found", skill: 1-impl }
  ```
- **Why It Matters:** First tool that treats **skill chaining as a first-class workflow primitive**. The LLM router pattern (using a separate model to decide routing based on skill output) is architecturally novel.

### 5. SkillWeave (curiosityexplorer/skillweave) — 0 stars (academic paper)

**Formal academic framework for governed multi-agent skill composition.**

- **Author:** Vivek Acharya (Boston University), submitted to IEEE Transactions on Services Computing
- **Core Contributions:**
  1. **Semantic Skill Composition Algebra (SSCA):** Formal model for skill compatibility, conflict detection, dependency resolution across agent boundaries
  2. **SkillWeave Orchestration Protocol (SWOP):** Extends MCP+A2A with skill negotiation, composition planning, governed execution contracts
  3. **Organizational Skill Governance (OSG):** Principal hierarchy enforcement, data boundary compliance, regulatory policies
  4. **Enterprise Skill Registry (ESR):** Governed marketplace with provenance, semantic versioning, access controls, audit trails
- **Testbed:** 12 agents (6 Financial Services, 6 Healthcare), 47 skills, 1,081 pairwise compositions
- **Baselines tested:** Ungoverned → Static Policy → Agent-Local → Full SkillWeave
- **Why It Matters:** First **formal academic treatment** of cross-agent skill composition. Addresses the critical gap: MCP connects agents to tools, A2A enables communication, but nothing governed **composition** until now. The healthcare + finance testbed validates real enterprise concerns (HIPAA, SOX compliance).

### 6. RuneHub (runehub-ai/rune-hub) — 5 stars

**Composable skill graph marketplace with DAG-based automation pipelines.**

- **What it is:** Registry of composable AI "skill graphs" called Runes
- **Structure:**
  - 139 service bundles (Gmail, Slack, ...) — `SKILL.json`
  - 228 atomic operations (gmail-fetch, slack-post, ...) — `SKILL.md`
  - 65 skill graphs / automation pipelines — `RUNE.json`
- **Architecture:** Three-layer abstraction:
  1. **Skills** — atomic operations
  2. **Skill Packages** — service bundles grouping related skills
  3. **Runes** — directed graphs composing skills into workflows
- **Tech:** Next.js frontend + CLI (`@rune-hub/cli`), built with Bun
- **Why It Matters:** First implementation of **skills as composable graph nodes** rather than standalone units. The Rune concept (named workflows from skill primitives) is a compelling abstraction.

### 7. x402guard (goheesheng/x402guard-web) — 0 stars

**Pre-install security auditing for AI agent skills with x402 crypto payments.**

- **Concept:** Security scanning service for skills before installation
- **Pricing Model:** x402 Protocol (HTTP 402 Payment Required) with USDC payments — 3 pricing tiers
- **Features:** Interactive skill audit simulator, trust stack diagram
- **Status:** Deprecated landing page, but the concept is significant
- **Why It Matters:** First attempt at **paid security auditing as a service** for the skill ecosystem. The x402 payment protocol integration is novel.

### 8. Diversio Agent Skills Marketplace (DiversioTeam/agent-skills-marketplace) — 2 stars

**Enterprise team marketplace following the emerging Agent Skills standard.**

- **Standard Compliance:** Follows agentskills.io/specification — the open, tool-agnostic format
- **Key Innovation — CI-enforced quality guardrails:**
  - SKILL.md hard limit: 500 lines max (CI fails above)
  - Progressive disclosure: metadata → full instructions → references
  - Portable across Codex + Claude Code
- **Plugins include:** Code review, atomic commits, PR workflows, ClickUp integration, MixPanel analytics, Bruno API docs
- **Why It Matters:** Best example of **enterprise-grade skill marketplace governance** with CI enforcement, size guardrails, and cross-platform compatibility standards.

## New Patterns Not Seen in Prior Rounds

### 1. Agent-to-Agent Commerce (SquidBay)
- Agents register identity, list skills, build reputation, get paid
- Three-tier pricing: rent (remote execution) / learn (skill file) / own (full package)
- Bitcoin Lightning micropayments with 2% platform fee
- Permanent identity — no renaming to escape bad reviews

### 2. Skill Orchestration as First-Class Primitive (skill-loop)
- YAML-defined skill chains with LLM-based routing
- Multi-agent runtime support (claude + codex + opencode in same workflow)
- Cron scheduling for autonomous recurring workflows
- Human-in-the-loop `blocked` states in orchestration graphs

### 3. Formal Composition Algebra (SkillWeave)
- Mathematical framework for skill compatibility and conflict detection
- Organizational governance integrated into composition lifecycle
- Cross-boundary security (HIPAA, SOX) enforcement at composition time
- Enterprise Skill Registry with provenance and audit trails

### 4. On-Chain Security Auditing (skill-audit-registry, x402guard)
- Solidity smart contracts on Base for immutable audit records
- x402 Protocol payments for security scanning services
- "Trust stack" approach: audit results stored on-chain, tied to skill versions

### 5. Hybrid Search for Skills (SkillX)
- Semantic embeddings (bge-base-en-v1.5) + keyword FTS5 + Reciprocal Rank Fusion
- Boost scoring from quality metrics and favorites
- Sub-second search latency at scale

### 6. Plugin Marketplace Protocol (Trail of Bits, SkillX, Diversio)
- `/plugin marketplace add owner/repo` — standardized distribution
- `marketplace.json` + `plugin.json` manifests
- Version pinning and update tracking
- Multi-plugin per marketplace

### 7. Standards Emergence
- **Agent Skills Specification** at agentskills.io/specification — open, tool-agnostic format
- SKILL.md with YAML frontmatter becoming the universal format
- Compatibility across 15+ platforms explicitly documented
- CI-enforced quality guardrails (500-line limits, naming conventions)

### 8. Composable Skill Graphs (RuneHub)
- Three-layer abstraction: atomic skills → service packages → workflow graphs (Runes)
- JSON-based DAG definitions for automation pipelines
- 228 atomic operations composable into 65 pre-built workflows

## Key Takeaways for Marketplace Design

1. **Search is critical:** SkillX's hybrid semantic+keyword search with <800ms latency shows skills need better discovery than browsing
2. **Three pricing tiers work:** SquidBay's rent/learn/own model elegantly handles different buyer needs
3. **Composition is the next frontier:** Both skill-loop (practical) and SkillWeave (academic) point to skill chaining as the killer feature
4. **Trust requires multiple layers:** Identity verification + on-chain audits + reputation systems + security scanning — no single approach suffices
5. **Standards are solidifying:** The SKILL.md + YAML frontmatter format is winning; agentskills.io/specification is the emerging standard
6. **Edge deployment matters:** SkillX on Cloudflare Workers shows the "npm for skills" model benefits from edge-first architecture
7. **Professional domains validate the model:** Trail of Bits finding real CVEs with security skills proves marketplace value beyond developer convenience
