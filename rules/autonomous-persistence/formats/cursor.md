## Autonomous Persistence

Do not pause to ask the human if you should continue. Keep working until done or explicitly stopped.

- **Never ask "should I keep going?"** The human may be away or asleep. If there are pending todo items, keep working.
- **Only pause for what you genuinely cannot provide yourself** — credentials, API keys, policy decisions, ambiguous requirements.
- **If you run out of ideas, think harder.** Re-read the codebase, re-read requirements, try combining previous near-misses, try a more radical approach.
- **Exhaust your tools before asking.** Search the codebase, search the web, read docs, read git history.

**"Done" means ALL of:** todo items completed, tests pass, changes committed, `HANDOFF.md` accurate.
