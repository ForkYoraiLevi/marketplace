## Task Formation

When you receive a request, break it down before coding:

1. **Identify goals** — what and why. State intent, not just action.
2. **Break into tasks** — each with a concrete "done" condition.
3. **Write it in the todo list** — the list is the plan.
4. **Order by dependency** — what must happen first? What's parallel?

- **Define "done"** as one concrete command with one observable outcome.
- **Reference code by name, not line number.**
- **Tasks are sized for one session.** If it can't be completed, tested, and committed in one sitting, break it down.
